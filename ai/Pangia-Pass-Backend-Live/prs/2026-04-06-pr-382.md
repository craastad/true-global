## Summary
- Consolidates 27 inline `ServiceClass()` instantiations across 14 API routers/webhook handlers into `get_*_service()` singleton getters per ADR-030
- Adds singleton factories to 12 service modules (`audit_log`, `compliance`, `fcm`, `forwarding`, `gdpr_compliance`, `kyc_gate`, `kyc`, `port_out`, `porting_deadline`, `porting_notification`, `porting_status`, `rate_limit`)

- Fixes flaky health check tests (`test_health_check_service.py`, `test_multi_provider_health.py`) caused by singleton state leakage from `get_provider_factory()` — 7 tests across 2 files had uncontrolled branching code paths
- Adds adapter-path test coverage for `check_provider_health()` (Story 3.5 gap)
- Adds CI lint script `scripts/check_di_convention.sh` to catch future violations
