# Pangia Pass - Repository Summary

## Overview

Pangia Pass is a SvelteKit 5 web application for managing global eSIM services with campaign-based promotions, partner affiliates, and subscription management. It serves as the customer-facing frontend for the Pangia Pass eSIM platform, providing dynamic landing pages, pricing, analytics dashboards, and payment flows.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Language | TypeScript 5.5 |
| Framework | SvelteKit 2.20 / Svelte 5.28 |
| Build | Vite 5.4 |
| Styling | SCSS (component-scoped, no Tailwind) |
| Auth | Firebase Authentication |
| Database | Firebase Realtime Database |
| Analytics | PostHog, Meta Pixel |
| Payments | Stripe (via backend APIs) |
| Hosting | Vercel |
| CI/CD | GitHub Actions |

## Key Features

- **Campaign Management**: Dynamic landing pages with per-campaign branding, pricing, and trial periods
- **Subscriptions**: Monthly, annual, lifetime plans with switching and extension
- **Affiliate Program**: Custom URLs, referral tracking, performance monitoring, special discounts
- **Partner Analytics**: Real-time campaign metrics, signups, conversions, geographic distribution
- **Payment Integration**: Stripe checkout via backend, coupon codes, payment method updates
- **Seasonal Events**: Site-wide promotions with priority system
- **CMS**: Staff-managed special affiliate discounts
- **Visualizations**: 3D globe (Globe.gl), coverage maps, Rive animations

## Database

Firebase Realtime Database:
- `/campaigns/{slug}/allowedUsers/{uid}` - Campaign access control
- `/analytics/campaigns/{slug}/stats` - Pre-computed campaign metrics
- `/users_secure/{uid}` - Secure user data
- `/cancellation/{uid}` - Cancellation state
- `/phone_numbers/{uid}`, `/user_names/{uid}`, `/marketing_consent/{uid}`

## API Routes

**Payment**: get-campaign-payment-url, switch-to-annual, switch-to-lifetime, extend-subscription-annual, get-update-payment-method-url

**Affiliates**: verify-affiliate, custom-affiliate-url-lookup, create-custom-url, get-uid-from-custom-url

**CMS**: special-affiliate-discounts (list, add, update-slug, remove)

**Campaigns**: Campaign management endpoints

**Utilities**: create-custom-og-description, update-tutorial

## Testing

- **Unit Tests** (Vitest): 17 tests, campaign auth business logic
- **Integration Tests** (Vitest + Firebase Emulator): 7 tests, SvelteKit load() functions
- **E2E Smoke Tests** (Playwright): 15 tests across 5 browsers, critical user journeys
- **Coverage**: 80-90% target, 100% pass rate
- **CI**: GitHub Actions pipeline (lint, type check, build, unit, integration, E2E)

## Deployment

- **Platform**: Vercel with `@sveltejs/adapter-auto`
- **Branches**: `main` (production), `staging`, `dev`
- **CI/CD**: GitHub Actions with concurrency control, Firebase emulators in CI
- **PostHog Proxy**: Vercel rewrites in `vercel.json`
