# Pangia Pass Backend - Repository Summary

## Overview

Pangia Pass Backend is a Python service powering the Pangia Pass eSIM platform. It orchestrates multi-provider eSIM provisioning, Stripe payment processing, usage tracking with analytics pipelines, campaign management, and admin dashboards. It runs on Google App Engine (Flask) and Cloud Run (FastAPI).

## Key Features

- **Multi-Provider eSIM**: Factory pattern supporting BICS, ESIMgo, Keepgo, PCCW, Tata, Transatel, Telnyx
- **Payments**: Stripe checkout, webhooks, subscription lifecycle (create/switch/extend/cancel), crypto support
- **Usage Pipeline**: Real-time tracking -> Cloud SQL staging -> batch aggregation -> analytics dashboards
- **Campaigns**: Dynamic pricing, transitions, renewals, coupon management
- **Referrals**: Custom checkout sessions, payout plans, affiliate discounts
- **Dashboards**: Usage insights, customer analytics, forecasting, affiliate performance, admin tools
- **Cron Jobs**: Hold release (5min), campaign transition (12hr), campaign renewal (6hr), PDP reset (3-10min), analytics, addon updates
- **Notifications**: Slack alerts (new users, failures, cancellations), Gmail emails

## Database

**Firebase Realtime Database** (primary):
- users, users_secure, locations, phone_numbers
- usages, usages_warnings_tracker, usage_aggregation
- referrals, referrals_payouts_plans
- surf_codes, campaign_coupons, special_affiliate_discounts
- customer_insights, bics_callbacks_data, latest_issued_esims

**Cloud SQL PostgreSQL** (analytics):
- Usage staging tables for aggregation pipeline
- Indexed usage data for dashboards
- Referral data sync
- Instance: `pangia-pass-live:europe-west1:pangia-pass-prod-usage`
